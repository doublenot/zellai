# zellai Schema

This document is the plugin architecture and data schema spec for zellai. It codifies component responsibilities, the status file format, the config schema, and the invariants yoyo must respect. The founding product vision lives in [`zellai-vision.md`](zellai-vision.md); this file is the practical schema that turns that vision into a disciplined implementation contract.

## Layers

**Plugin layer** lives in `src/`. The Zellij plugin binary (`zellai.wasm`) renders the sidebar, reads status files, and manages workspace state. It implements the `ZellijPlugin` trait.

**Hook layer** lives in `hooks/`. Shell scripts invoked by Claude Code (and other agents) to write status files. These are the write side of the status bridge.

**Wrapper layer** handles agents that don't support native hooks. `zellai run` and named wrappers (`zellai-codex` etc.) wrap the agent process and write status on its behalf.

**Schema** is this file. It co-evolves with the project: when yoyo adds a new component, event type, or convention, it updates this document in the same commit.

## Status File Schema

Status files are the IPC layer between agents and the plugin. Each active agent session writes one file.

**Location:** `<user-data-dir>/zellai/sessions/<session-id>.json`

**Session ID:** `<hostname>-<zellij-pane-id>` or a UUID generated by the wrapper at startup.

**Schema:**

```json
{
  "version": 1,
  "session_id": "hostname-pane-42",
  "agent": "claude",
  "status": "thinking",
  "git_branch": "feat/auth",
  "git_dirty": false,
  "working_dir": "/home/user/projects/app",
  "last_message": "Reading src/auth.ts…",
  "ports": [3000, 5173],
  "pr_number": 42,
  "pr_ci_status": "passing",
  "needs_attention": false,
  "updated_at": 1706000101
}
```

**Field definitions:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `version` | integer | yes | Schema version. Always `1` for now. |
| `session_id` | string | yes | Unique ID for this agent session. |
| `agent` | string | yes | Agent name: `"claude"`, `"codex"`, `"gemini"`, `"aider"`, `"opencode"`, `"unknown"` |
| `status` | string | yes | One of: `"thinking"`, `"waiting"`, `"idle"`, `"error"` |
| `git_branch` | string or null | yes | Current git branch, or `null` if not in a git repo |
| `git_dirty` | boolean | yes | Whether the working tree has uncommitted changes |
| `working_dir` | string | yes | Absolute path of the agent's working directory |
| `last_message` | string or null | yes | Most recent status line or notification from the agent |
| `ports` | array of integers | yes | Listening TCP ports detected in the pane (empty array if none) |
| `pr_number` | integer or null | no | Linked PR number from `gh` CLI, or `null` |
| `pr_ci_status` | string or null | no | `"passing"`, `"failing"`, `"pending"`, or `null` |
| `needs_attention` | boolean | yes | Whether the agent is waiting for human input |
| `updated_at` | integer | yes | Unix epoch seconds of last update |

**Status values:**

- `"thinking"` — agent is actively processing, no human input needed
- `"waiting"` — agent has paused and needs human input (triggers attention indicators)
- `"idle"` — agent finished its task, not running
- `"error"` — agent encountered an unrecoverable error

**Invariants:**
- `needs_attention` must be `true` if and only if `status == "waiting"`
- `updated_at` must advance monotonically within a session
- The plugin treats a file not updated within 60 seconds as stale and marks the agent `"idle"`
- Files are deleted when a session ends (hook: on-stop)

## Config Schema (`zellai.toml`)

`zellai.toml` is optional. All values have defaults. The plugin loads from the current directory upward (walking parent directories until found or repo root reached), then falls back to `<user-config-dir>/zellai/zellai.toml`.

```toml
[sidebar]
position = "left"           # left | right | bottom
card_density = "adaptive"   # compact | detailed | adaptive
attention_animation = true  # false disables pulsing glow

[teams]
default_layout = "orchestrator-top"   # orchestrator-top | orchestrator-left | equal-grid | custom
orchestrator_agent = "claude"         # default agent for orchestrator pane
worker_agent = "claude"               # default agent for worker panes
worker_count = 2                      # default number of worker panes

[bridge]
sessions_dir = "~/.local/share/zellai/sessions"   # status file directory
poll_interval_ms = 500                             # how often to re-read status files
stale_threshold_s = 60                             # seconds before marking agent idle

[keybindings]
next_attention = "Ctrl a"   # cycle to next pane needing attention
dismiss = "Ctrl d"          # dismiss current pane notification
jump_to = "Ctrl g"          # open pane picker
```

**`[[teams.layout]]` custom layout blocks:**

```toml
[[teams.layout]]
name = "my-team"
panes = [
  { role = "orchestrator", agent = "claude", prompt = "You are coordinating the team." },
  { role = "worker", agent = "codex", prompt = "Implement the auth module." },
  { role = "worker", agent = "gemini", prompt = "Write tests for the auth module." },
]
```

## Component Responsibilities

### `src/main.rs` — Plugin Entry Point

Implements `ZellijPlugin`. Owns the top-level event loop.

- `load()` — initialise state, subscribe to `FileSystemUpdate`, `Timer`
- `update()` — dispatch events to sub-components
- `render()` — delegate to `sidebar.rs`; must return immediately (non-blocking)

### `src/sidebar.rs` — Sidebar Renderer

Owns the visual output of the plugin pane.

- Renders the list of agent cards using current `AgentStatus` state
- Implements adaptive card density: expands `needs_attention == true` cards, collapses others
- Renders attention indicators: badge dot + ambient glow for waiting agents, dimming for idle agents
- Respects `sidebar.position` and `sidebar.card_density` from config

### `src/status_bridge.rs` — Status File Reader

Owns reading and parsing the `<user-data-dir>/zellai/sessions/` directory.

- Called on `FileSystemUpdate` events (watch via Zellij API) and on `Timer` tick
- Parses each `.json` file into an `AgentStatus` struct
- Marks stale sessions (last `updated_at` > `stale_threshold_s`) as `"idle"`
- On parse, if `status != "waiting"`, forces `needs_attention` to `false` regardless of file contents (defensive guard against race between hook write and file delete)
- Deletes state for sessions whose files have been removed

### `src/config.rs` — Config Parser

Owns loading and validating `zellai.toml`.

- In v1: loads only from `<user-config-dir>/zellai/zellai.toml` (user-global). CWD-based directory walking is unreliable inside a WASM plugin — the plugin's CWD is the Zellij host's working directory, not the user's terminal CWD. Project-local config discovery is deferred until a pane-CWD mechanism is defined.
- Returns a `ZellaiConfig` struct with all fields populated (defaults filled in for missing keys)
- Never fails on missing file — always returns defaults

### `src/workspace.rs` — Workspace Manager

Owns named workspace save/restore.

- Persists workspace layouts to `<user-data-dir>/zellai/workspaces/<name>.json`
- `save(name)` — serialise current pane layout + agent assignments
- `restore(name)` — reconstruct the layout, launch agents

### `src/teams.rs` — Teams Launcher

Owns the `zellai teams` entry point.

- Reads `[teams]` config from `zellai.toml`
- Constructs the pane layout (default: orchestrator-top)
- Opens panes via Zellij API, runs the configured agent command in each

### `src/attention.rs` — Attention State

Owns the cross-component attention state.

- Tracks which sessions have `needs_attention == true`
- Provides `next_attention()` for keyboard navigation cycling
- Provides `dismiss(session_id)` to clear attention for a pane

### `hooks/` — Claude Code Hook Scripts

Shell scripts registered with Claude Code via `zellai init`.

- `on-stop.sh` — writes `status: "idle"` AND `needs_attention: false` (must set both together), then deletes the status file on clean exit
- `on-notification.sh` — writes `needs_attention: true`, `last_message` from notification
- `on-post-tool-use.sh` — updates `last_message` with current tool name, `status: "thinking"`

All hooks receive the session ID as `$ZELLAI_SESSION_ID` (set by the plugin at pane creation via environment variable injection).
